# Лендинг — Илья Петров · AI Engineer / Prompt Engineer

Сайт-портфолио под профиль вакансии **AI Engineer / Prompt Engineer**.
Цель — не пересказать резюме, а **аргументировать соответствие роли**: прямой
маппинг «что нужно роли → где я это делал в production».

🔗 **Живая версия:** _(добавить ссылку после деплоя)_

---

## Что внутри

| Секция | Зачем |
|---|---|
| **Hero** | Позиционирование + 4 ключевые метрики (20 скиллов, 150–200 чел-ч/мес, ×16–×240). |
| **Почему я** | Маппинг требований роли на конкретный опыт — ядро лендинга. |
| **Технологии** | Релевантный стек, сгруппированный по граням роли (LLM, агенты, RAG, ETL, оценка). |
| **Проекты** | Production-агент (КАМА), Resume Scorer, RAG/инфра пет-проекты. |
| **CV-трек** | Computer Vision как вторичная релевантная компетенция. |
| **Контакты** | Email, Telegram, GitHub. |

Двуязычный интерфейс **RU / EN** (переключатель в шапке; язык запоминается в
`localStorage`, по умолчанию — из языка браузера).

## Стек

- **React 19 + TypeScript** (Vite)
- **Tailwind CSS v3** — дизайн-система в `tailwind.config.js`
- Контент вынесен в типизированный словарь `src/content/{ru,en}.ts` —
  правки текста без касания компонентов.
- Анимации появления на чистом `IntersectionObserver` (без зависимостей).

```
src/
  content/        # i18n-словарь (ru.ts, en.ts) + типы — единственный источник текста
  components/     # Nav, Hero, RoleFit, Tech, Projects, CvTrack, Contact, Section, Reveal
  App.tsx         # сборка секций + выбор языка
```

## Запуск

```bash
npm install
npm run dev      # http://localhost:5173
npm run build    # сборка в dist/
npm run preview  # предпросмотр прод-сборки
```

## Деплой

Готов к статическому хостингу из коробки:

- **Vercel** — `vercel.json` (build `npm run build`, output `dist`, SPA-rewrite).
- **Netlify** — `netlify.toml` (та же сборка + SPA-redirect).

Любой из них: импортировать репозиторий → деплой без доп. настройки.

## Решения

- **Контент отделён от вёрстки** — текст в `content/`, что упрощает правки и
  держит обе локали синхронными по структуре (общий тип `Content`).
- **Tailwind v3, не v4** — стабильный классический workflow с `postcss`.
- **Без тяжёлых анимационных библиотек** — `IntersectionObserver` достаточно,
  бандл лёгкий (~69 КБ gzip).
